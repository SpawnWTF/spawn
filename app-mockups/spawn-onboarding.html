Clawdbot Guide

‚Äπ Docs
Connect Clawdbot
‚ñ∂

Watch: Connect Clawdbot in 2 minutes

‚ÑπÔ∏è
Prerequisites: Clawdbot running on your Mac, Linux server, or Docker container.
1
Create an agent in the app
Tap the + button and select "Connect Existing Agent". Give it a name and copy your token.
2
Add token to your environment
Add to your .env file or export directly:
# In ~/.clawdbot/.env SPAWN_TOKEN=spwn_sk_your_token
3
Install the Spawn skill
cd ~/clawdbot/skills npx spawn-skill init
This creates the skill folder and configures the integration.
4
Restart Clawdbot
# If running directly: clawdbot restart # If using Docker: docker restart clawdbot
‚úì Your agent should appear in the app!

Common Questions

What if I don't have a skills folder? ‚Ä∫
You can add the integration directly to your main script instead. See the "Manual Integration" section below.
Will this affect my existing setup? ‚Ä∫
No. Spawn is additive ‚Äî it doesn't change how Clawdbot runs. It just adds a new communication channel.
Can I still use Telegram? ‚Ä∫
Yes! You can use both simultaneously. Many users keep Telegram for quick commands and use Spawn for complex tasks.
Is my data secure? ‚Ä∫
Messages are end-to-end encrypted. Your agent token is like a password ‚Äî never share it. You can regenerate it anytime in Settings.

from spawn import ui, status, approval, policy async def do_risky_thing(path): # Check policy first if policy.is_path_forbidden(path): await ui.send_text(f"Can't access {path}") return # Ask for approval if policy.requires_approval("files.write"): ok = await approval.confirm( title=f"Write to {path}?", danger_level="medium" ) if not ok: return # Do the thing await status.set("working", "Writing...") write_file(path) await status.set("idle") await ui.send_text("Done!")
